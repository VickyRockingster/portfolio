Scribe

Tools/Technologies Used:
Axios CSS3 Git GitHub Heroku HTML5 JavaScript Material UI PostgreSQL React.js Ruby on Rails Sass/SCSS

I find journaling to be great for relaxing and processing at the end of the day, so I wanted to create an application that would allow me to do that online. Scribe is a single-page-application that allows users to write journal entries in an online application.
In the back-end, I thought to use a Rails api with a non-relational database. However, I figured that this app could potentially be used to take notes, write books, or otherwise engage in a writing activity that would require building out relationships beyond user and journal entries. For that reason, I decided to use the relational database PostgreSQL instead.
As for the front-end, I used React.js. I wanted to challenge myself to learn more about different kinds of web tools that can accomplish the same goal using different technologies and methods than the JavaScript and jQuery I was used to.

ListNote

Tools/Technologies Used:
AJAX Bootstrap CSS3 Git GitHub Handlebars Heroku HTML5 JavaScript jQuery PostgreSQL Ruby on Rails Sass/SCSS

I created ListNote because I love making lists, but I've had some frustration in not finding an application that differentiated between different kinds of to-do lists. ListNote is a single-page application that addresses that issue.
Users can create a 'chores' to-do list, that lists all the things the user needs to do around the house (e.g. do laundry), or an 'errands' to-do list, that lists all the things the user needs to do outside the house (e.g. pick up dry-cleaning).
Moving forward, I plan to build out features that would allow signed-in users to share lists, as well as create their own. This would require some re-factoring to create a many-to-many relationship between the user and each of the other resources (i.e. errands, chores) in the back-end, as well as building out that functionality in the front-end.
